# Карта scope

Этот документ фиксирует, что именно строится в каком порядке. Ничего из идеи не
выкинуто: фичи разложены по этапам, чтобы разработка не утонула в маскоте,
инвентаре и визуальных эффектах до появления рабочего ядра задач.

## MVP-0. Рабочее ядро

Цель: получить приложение, в котором можно реально вести задачи без сервера.

Входит:

- Nuxt/Vue shell приложения;
- клиентский мобильный runtime через Capacitor;
- локальная SQLite БД;
- миграции БД;
- первый запуск и локальный профиль по умолчанию;
- главный экран;
- создание задачи с одним обязательным полем `title`;
- редактирование задачи;
- архивация задачи вместо физического удаления;
- выполнение задачи;
- дедлайн задачи;
- приоритет задачи;
- автоматическое предположение сложности задачи;
- ручное изменение сложности;
- сортировка и группировка задач;
- вычисляемая просрочка;
- XP за выполнение задач;
- повышение уровня;
- статичная темная базовая тема;
- базовые design tokens: цвета, отступы, радиусы, типографика;
- сохранение данных после перезапуска;
- unit-тесты доменной логики.

Не входит в MVP-0:

- маскот как визуальный персонаж;
- инвентарь;
- экипировка предметов;
- локальные уведомления;
- визуальная рандомизация;
- настройки;
- REST API;
- синхронизация.

## MVP-1. Маскот, инвентарь и XP-множители

Цель: добавить легкую геймификацию, которая усиливает выполнение задач, но не
перегружает основной task flow.

Входит:

- полноценный экран профиля;
- маскот;
- каталог базовых предметов;
- выдача предметов за уровни;
- защита от повторной выдачи одной и той же награды за уровень;
- случайная выдача предметов после выполнения задачи по XP;
- защита от повторного reward-roll для одной и той же задачи;
- ролл характеристики предмета при выпадении;
- XP-множитель как единственная характеристика предмета в MVP-1;
- инвентарь;
- экипировка предметов;
- применение XP-множителей только от надетых предметов;
- настраиваемые слоты маскота с anchor points;
- отображение предметов поверх маскота;
- базовая статистика пользователя.

Ограничение: в MVP-1 предметы дают только XP-множитель. Бонусы к шансам
выпадения, штрафам, скорости, сериям и другим механикам не добавляются.

## MVP-2. Напоминания, визуальная вариативность, настройки

Цель: добавить удержание внимания и удобство без ломки стабильной структуры.

Входит:

- локальные уведомления для задач с дедлайном;
- request permissions;
- отмена уведомления при выполнении/архивации;
- пересоздание уведомления при изменении дедлайна;
- визуальные варианты поверх статичной темной базы;
- controlled visual random;
- сохранение visual state;
- варианты текста главной кнопки;
- варианты заголовка списка;
- фон профиля;
- настройки;
- `reducedMotion`;
- `disableVisualRandomness`;
- отключение звуков/эффектов, если они появятся.

## Post-MVP

Фичи, которые сохраняются как будущие направления, но не блокируют дипломный
MVP:

- регистрация;
- авторизация;
- сервер;
- REST API как основной источник данных;
- синхронизация между устройствами;
- командные задачи;
- комментарии;
- вложения файлов;
- полноценный календарь;
- канбан-доски;
- привычки;
- Pomodoro;
- сложные повторяющиеся задачи;
- магазин предметов;
- донат;
- социальные функции;
- ИИ-помощник;
- бонусы предметов к шансам выпадения;
- бонусы предметов кроме XP-множителя;
- штрафы/смягчение штрафов;
- медицинские рекомендации.

## Правило изменения scope

Новая фича добавляется только в один из этапов:

```txt
MVP-0 -> MVP-1 -> MVP-2 -> Post-MVP
```

Если фича не нужна для создания, просмотра и выполнения задач, она не попадает в
MVP-0.
