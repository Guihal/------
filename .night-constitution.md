# Night Mode Constitution

## Hard Invariants
- **Operator**: только `docs/**` + repo-root `*.md`. Код — строго через implementer subagent.
- **Executor**: только `allowed_write_paths` из packet. Вне путей = audit break.
- **Architect**: read-only, вердикты по инвариантам (spec alignment).
- **Critic**: read-only, только блокеры (секреты, деструктив, edge-cases).
- **MAX_ROUNDS=5** per packet. Round 6 → auto-escalate (architect verdict).
- **Strategic review** every 3 committed packets (SR01 after Q03, SR02 after Q06...).
- **Spec drift** → правится spec, не реализация.
- **Bounded context**: executor видит packet + read_context, не full spec.
- **Parallel cap**: max 3 active subagents.

## Platform Wave Canon
- **Server**: `workspace/server`, Bun + PostgreSQL, source of truth.
- **Admin**: `workspace/admin-panel`, отдельное web-приложение.
- **App**: `workspace/app`, клиент, только отображает server state.
- **Auth**: Bearer token, NO cookies. Roles: user/admin.
- **Server owns**: XP, levels, items, rarity, drop roll, inventory, logs, stats.
- **SQLite removed** from domain persistence.
- **OpenAPI contract**: `workspace/server/openapi.json`.
- **Packet order**: `docs/specs/platform-qwen-packets.md`.

## Execution Rules
- Не менять файлы вне `allowed_write_paths` текущего packet.
- Не переписывать старые ADR (только append).
- При неоднозначности → субагент-архитектор решает, не юзер.
- Diff budget и file_count_max из packet — hard limits.
- Каждый тик: read state → execute → verify → save state.
- Никогда не угадывать архитектуру. Packet = contract.
- Если packet требует решения не из spec → эскалация на архитектора.
- После каждого тика: обновить `.night-state.json`.

## Escalation Flow
- Packet escalation trigger fired → architect subagent анализирует.
- Architect verdict: `{ fix_target: impl|spec|both|escalate, issues[] }`.
- `fix_target=impl` → feedback implementer, retry.
- `fix_target=spec` → operator patches docs, retry.
- `fix_target=escalate` → записать в `state.errors`, skip packet, continue.
